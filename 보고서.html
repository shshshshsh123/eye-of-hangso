한 줄 소개
플레이어는 판타지 세계의 유물 감정관이 되어, 의뢰서에 적힌 정보와 실물·규정집을 대조하며 들어오는 유물이 진품인지 위조품인지 판별한다. 단 하나의 모순이라도 발견되면 반려해야 하며, 일주일 단위로 봉급과 벌금이 정산되는 규칙 기반 판단(Papers, Please 계열) 게임이다.

1프로젝트 개요

1.1 작품 주제 선정 배경

"화려한 액션이 아니라, 관찰하고 판단하는 행위 자체를 게임으로 만들 수 없을까?"

이 프로젝트는 단순 반사신경형 게임이 아니라, 플레이어의 주의력과 규칙 추론을 핵심 재미로 삼는 게임을 목표로 시작했다. 모티프가 된 Papers, Please는 '서류 검사'라는 지극히 평범한 행위에 도덕적 긴장과 규칙의 누적을 얹어 깊은 몰입을 만들어낸다. 우리는 여기에 누구나 흥미를 느낄 판타지 유물 감정이라는 옷을 입혔다.

플레이어는 왕도의 유물 감정소에서 일하는 감정관이다. 매일 여러 점의 유물이 의뢰서와 함께 들어오고, 감정관은 의뢰서의 기재 사항(재질·각인·출처·마력 상태 등), 실물 외형, 그리고 규정집을 교차 검증하여 매입 승인 여부를 결정한다. 출처와 각인이 어긋나거나, 마력이 소진된 무기가 섞여 들어오거나, 의뢰서의 품명과 실물 모양이 다른 등 위조의 단서를 찾아내는 것이 핵심 플레이다.

날이 지날수록 위조 수법이 정교해지고(단순 위조 마크 → 복합 위조), 잘못된 판정에는 벌금이 부과되므로 자연스럽게 난이도 곡선이 형성된다. "규칙을 이해할수록 더 잘 보이는" 학습형 재미를 의도했다.

2.1 사용 파일 구성도

프로젝트는 씬(.tscn) · 스크립트(.gd) · 데이터(.txt) · 리소스로 역할을 분리하여 구성했다.

EyeOfHangso/
├─ Scenes/
│  ├─ MainMenu.tscn     # 타이틀 화면 (시작 씬)
│  └─ WeeklySummary.tscn # 주간 결산 화면
├─ main_game.tscn       # 핵심 감정 플레이 화면
├─ Scripts/
│  ├─ GameManager.gd     # [AutoLoad] 점수·일수·저장·주간정산
│  ├─ AudioManager.gd    # [AutoLoad] BGM/SFX 볼륨 관리
│  ├─ SceneTransition.gd # [AutoLoad] 씬 전환 페이드
│  ├─ MainGame.gd       # 유물 생성·판정·UI 연출
│  ├─ MainMenu.gd       # 메뉴 버튼·옵션 슬라이더
│  ├─ WeeklySummary.gd   # 결산 항목 연출
│  └─ ArtifactData.gd    # 유물 데이터 클래스 + Enum 정의
├─ Data/
│  └─ ArtifactData.txt   # 유물 50종 정답 DB (CSV)
├─ Resource/
│  ├─ Artifact/AtlasTextures/ # 유물 스프라이트 50종(.tres)
│  ├─ Audio/          # BGM 1 + 효과음 5
│  └─ (배경·버튼·책 등 UI 이미지)
├─ Font/ BMJUA_ttf.ttf # UI 폰트
└─ project.godot       # 엔진 설정·AutoLoad 등록
파일 핵심 책임
ArtifactData.gd 유물 한 점을 표현하는 데이터 클래스. 재질/각인/분류/색상/등급/출처/마력상태/특이규칙을 Enum으로 정의하여 코드 전반에서 타입 안전하게 참조.
MainGame.gd 게임의 심장부. CSV 로드 → 진품/위조품 절차적 생성 → 의뢰서·룰북·실물 검사 UI → 판정 채점 → 다단계 결과 연출까지 담당.
GameManager.gd 씬이 바뀌어도 유지되는 전역 상태(점수·일수·주간 통계·최고기록). 판정 반영(+10/−5), 7일 주기 결산 계산, ConfigFile 기반 최고기록 저장.
SceneTransition.gd CanvasLayer(layer=128)로 항상 최상단에 떠서 씬 교체 순간의 깜빡임 없이 검정 페이드 전환을 제공.
AudioManager.gd 스트림 사전 로드 + 버스 볼륨 제어. 클릭/정답/오답/책장 넘김 효과음을 단축 메서드로 노출.

2.2 UI / UX 설계

"책상 위에서 서류를 이리저리 펼쳐 보는" 감각을 살리기 위해, 주요 창(의뢰서·규정집·실물 검사)을 모두 드래그로 자유롭게 옮길 수 있는 윈도우로 구현했다. 여러 창이 겹치면 클릭한 창이 맨 앞으로 올라오며, 플레이어가 동시에 여러 단서를 비교하도록 유도한다.

① 메인 메뉴

게임 시작 / 옵션 / 종료. 최고 기록을 상단에 노출하고, 옵션에서 배경음·효과음 볼륨을 슬라이더로 조절한다.

② 의뢰서 패널

유물의 품명과 기재 정보(재질·각인·분류·색상·희귀도·출처·마력 상태)를 표시. 특이 규칙이 있으면 ⚠ 경고로 강조한다.

③ 실물 검사 팝업

버튼을 누르면 유물의 실제 외형 이미지를 띄운다. 의뢰서 품명과 외형이 다르면 그 자체로 위조 단서다.

④ 규정집(룰북)

출처별 허용 각인·금지 재질, 마력 상태 규정, 분류·등급별 규칙, 그리고 유물 도감(이미지 카탈로그)을 페이지로 제공.

⑤ 판정 & 결과 연출

승인/반려 후 "판정 중…" 서스펜스 → 정/오답 → 사유 → 점수 변동 → 주간 현황을 단계별 페이드로 보여준다.

⑥ 주간 결산

7일마다 봉급·벌금·순이익을 항목별로 순차 공개. 최고 기록 갱신 시 별도 표식을 띄운다.

2.3 핵심 로직 다이어그램

(가) 게임 플레이 루프 & 위조품 절차적 생성

위조 변형은 단순히 무작위가 아니라 일수 기반 가중치 테이블로 조절된다. 초반에는 조잡한 위조 마크처럼 알아보기 쉬운 기믹의 가중치가 높고, 일수가 오를수록 점차 낮아진다. 반대로 2일차에는 '미상 출처 + 공식 각인' 기믹이, 3일차에는 '복합 위조'·'동일 분류 내 이미지 교체' 같은 고난도 기믹이 해금되어 가중치가 누적 상승한다.

(나) 채점 및 주간 경제 규칙

상황 처리
판정 정답 점수 +10, 주간 정답 건수 +1
판정 오답 점수 −5, 주간 오답 건수 +1 (점수는 0 미만 방지)
주간 봉급 (7일 정산) 주간 정답 1건당 +5
주간 벌금 (7일 정산) 주간 오답 1건당 −3
최고 기록 정산 점수가 최고치 초과 시 ConfigFile로 영구 저장
진품인데 반려해야 하는 함정
"진짜 유물이지만 규정상 매입 불가"인 경우가 존재한다. 예컨대 마력이 소진(DEPLETED)된 무기·물약은 진품이어도 매입 규정 위반이다. 이를 위해 생성 직후 ValidateGenuineArtifact()가 진품 결과를 한 번 더 검증해, 규정 위반 진품을 자동으로 '반려가 정답'으로 보정한다.

3테스트·검증 및 개발 중 문제 해결

3.1 테스트 및 검증 방법

  • 판정 로직 검증 — 각 위조 기믹이 의도한 규칙 위반을 실제로 만들어내는지, 콘솔에 유물명·진품 여부·위조 사유를 출력해 수십 회 반복 확인.
  • 경계값 테스트 — 점수가 0 미만으로 떨어지지 않는지, 주간 정산이 정확히 7일(5점×일)마다 트리거되는지 검증.
  • 저장/불러오기 — 게임 재실행 후 최고 기록이 유지되는지 확인.
  • UI 상호작용 — 창 드래그·겹침 우선순위, ESC 동작(옵션 열기/팝업 닫기), 연속 클릭 방지(waiting_for_next 가드) 점검.
  • 플레이 밸런스 — 초반에 너무 어렵지 않고 후반에 충분히 까다로운지, 가중치 테이블을 반복 조정하며 체감 난이도 확인.

3.2 개발 중 발생한 장애요인과 해결 과정

문제 1 · 씬 전환 시 화면 깜빡임

증상: 씬을 교체하는 순간 한 프레임 검은 화면/흰 화면이 번쩍였다.
원인: 페이드 연출 노드가 교체되는 씬에 종속되어, 씬이 사라질 때 함께 파괴됨.
해결: 전환 오버레이를 CanvasLayer(layer=128)의 AutoLoad(SceneTransition)로 분리해 씬 교체와 무관하게 항상 살아있도록 만들고, "검정 페이드 인 → 씬 교체 → 다음 프레임 페이드 아웃" 순서로 처리하여 깜빡임을 제거.

문제 2 · 창을 Panel 밖으로 빠르게 끌면 드래그가 끊김

증상: 의뢰서·룰북 창을 마우스로 빠르게 움직이면 커서가 패널 영역을 벗어나며 드래그가 풀렸다.
원인: 각 패널의 gui_input에 의존하면 커서가 노드 밖으로 나가는 순간 이벤트가 끊김.
해결: 드래그를 개별 gui_input이 아니라 전역 _input()에서 일괄 처리하도록 변경. 클릭 시점에 겹친 창 중 최상단을 골라 드래그 대상으로 잡고, 마우스 이동/릴리즈를 전역에서 추적해 패널 밖으로 나가도 끊기지 않게 했다.

문제 3 · 위조 기믹이 의도치 않게 '이중 위반'이 됨

증상: 출처만 바꾸려 했는데 각인까지 어긋나 단서가 두 개가 되어, 의도한 난이도보다 쉬워지거나 사유가 모호해졌다.
원인: 출처를 변형하면서 기존 각인이 새 출처의 규정과 자동으로 충돌.
해결: 각 기믹에서 변형 대상 외의 속성은 새 규정에 맞게 보정(예: 왕도 출처로 바꿀 때 각인은 ROYAL로 맞춰 위반을 '재질' 한 가지로 한정)하여 단일 위반을 보장. 복합 위조는 별도 기믹으로 명시 분리.

문제 4 · 진품인데 반려가 정답인 케이스 누락

증상: 마력이 소진된 무기가 '진품'으로 분류되어, 규정대로 반려한 플레이어가 오답 처리됨.
해결: 생성 직후 ValidateGenuineArtifact()를 거쳐 DEPLETED 무기·물약 등 규정 위반 진품을 자동으로 '반려가 정답'으로 보정하는 후처리 단계를 추가.

문제 5 · 결과 팝업 애니메이션 노드 잔류

증상: 빠르게 다음 유물로 넘기면 이전 판정 연출용 라벨이 화면에 남았다.
해결: 결과 연출 노드에 _anim_ 접두어 규칙을 부여하고, 팝업을 새로 띄울 때 해당 노드를 모두 정리. 연출 중 노드가 파괴될 수 있으므로 매 await 뒤 is_instance_valid()로 유효성을 확인해 오류를 방지했다.

4성과 및 결론

4.1 기대효과 및 활용방안

  • 규칙 학습형 게임의 교육적 활용 — 규정을 읽고 단서를 대조해 결론을 내리는 구조는 추론·주의력 훈련 콘텐츠로 응용할 수 있다.
  • 콘텐츠 확장성 — 유물 DB가 CSV 한 장으로 분리되어 있어, 코드 수정 없이 유물·규칙을 추가하며 분량을 늘릴 수 있다.
  • 데이터 주도 설계의 템플릿 — "데이터(규칙) + 절차적 변형 + 판정" 구조는 다른 검사·심사형 게임(통관, 면접, 품질검사 등)으로 재활용 가능한 틀이다.
  • 난이도 자동 조절 — 일수 기반 가중치 방식은 별도 레벨 디자인 없이도 자연스러운 난이도 곡선을 제공한다.

4.2 프로젝트 성과

콘텐츠 분량 유물 50종 DB, 위조 기믹 15종, 일수 기반 난이도 가중치

4.3 후기

이번 프로젝트에서 가장 크게 배운 것은 "재미는 규칙의 설계에서 나온다"는 점이었다. 화면을 화려하게 꾸미는 것보다, 위조 단서를 어떻게 숨기고 드러낼지, 난이도를 어떤 속도로 올릴지를 고민하는 과정이 게임의 완성도를 좌우했다. 가중치 테이블 하나를 바꾸는 것만으로 체감 난이도가 크게 달라지는 경험을 통해, 데이터 주도 설계의 힘을 체감했다.

엔진 측면에서는 AutoLoad 싱글톤으로 씬 간 상태를 유지하고, CanvasLayer로 전환 연출을 분리하는 등 Godot의 구조적 패턴을 익혔다. 또한 드래그·연출처럼 사소해 보이는 UX 디테일이 실제로는 상당한 예외 처리를 요구한다는 사실을, 여러 버그를 잡으며 직접 깨달았다.

아쉬운 점은 위조 사유의 시각적 강조(실물 위 하이라이트)나 스토리 라인을 충분히 넣지 못한 것이다. 향후 유물·규칙을 확장하고, 판정 통계 기반의 엔딩 분기를 더한다면 더 깊이 있는 작품으로 발전시킬 수 있을 것이라 본다.
